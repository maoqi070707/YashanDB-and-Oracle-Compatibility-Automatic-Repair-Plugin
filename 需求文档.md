# YashanDB Oracle兼容修复工具需求文档

## 1. 项目概述

YashanDB Oracle兼容修复工具是一个专门用于将Oracle SQL语句转换为YashanDB兼容格式的工具，旨在解决Oracle数据库迁移到YashanDB时的SQL兼容性问题。

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 SQL扫描与分析
- 自动扫描Oracle SQL语句中的兼容性问题
- 识别需要修复的Oracle特有语法
- 对问题进行分级（严重、中等、轻微）

#### 2.1.2 SQL修复
- 自动将Oracle SQL转换为YashanDB兼容格式
- 支持函数、语法、类型等多种修复规则
- 提供详细的修复建议

#### 2.1.3 SQL验证
- 连接YashanDB验证修复后的SQL是否能正常执行
- 提供验证结果和执行耗时
- 针对验证失败的情况提供详细错误信息

#### 2.1.4 文件处理
- 支持单个SQL文件上传和处理
- 支持批量SQL文件上传和处理
- 提供文件处理进度和结果汇总

### 2.2 可视化功能

#### 2.2.1 SQL处理结果可视化
- 拆分视图展示：左侧原始Oracle SQL，右侧修复后YashanDB SQL
- 语法高亮：SQL关键字、函数、类型标色
- 差异对比标注：用红色标注需要修复的部分，绿色标注修复后的内容
- 问题分级提示：扫描结果按严重程度分级，用不同颜色标签展示

#### 2.2.2 批量处理进度可视化
- 批量上传/处理文件时，显示进度条（百分比 + 剩余时间预估）
- 每个文件处理状态用图标区分：✅成功、❌失败、⌛处理中、⚠️有警告
- 处理完成后展示汇总面板：总文件数、成功数、失败数、警告数
- 支持导出处理报告

#### 2.2.3 修复规则可视化映射表
- 以交互式表格展示所有支持的修复规则
- 包含「Oracle 语法」「YashanDB 语法」「适用场景」「示例」四列
- 支持按「函数 / 语法 / 类型」分类筛选，支持关键词搜索
- 点击规则可展开详细示例（输入 / 输出 SQL 对比）

#### 2.2.4 SQL验证结果可视化
- 验证通过时展示绿色对勾 +「验证成功」提示，附带执行耗时
- 验证失败时展示红色叉号，展开面板显示具体错误信息
- 关联对应的修复规则建议
- 支持将验证结果导出为文本 / 截图

#### 2.2.5 操作日志可视化
- 记录用户的核心操作，以时间轴形式展示
- 每条日志包含「操作类型、操作时间、操作结果、关联 SQL / 文件名称」
- 支持按时间范围、操作类型筛选日志
- 支持清空 / 导出日志

#### 2.2.6 数据统计可视化仪表盘
- 展示累计处理 SQL 条数、文件数、修复成功率、高频修复规则 TOP10
- 以饼图展示问题分级占比（严重 / 中等 / 轻微）
- 以柱状图展示不同类型修复规则的使用频次
- 支持按日 / 周 / 月筛选统计数据

#### 2.2.7 拖拽式批量文件上传可视化
- 支持拖拽文件到指定区域，拖拽时显示悬浮提示
- 上传区域实时显示已选择文件的列表（文件名、大小、类型）
- 支持删除单个文件

#### 2.2.8 修复流程引导可视化
- 首次使用时展示分步引导（输入 SQL→扫描→修复→验证）
- 以动态箭头 + 高亮提示引导操作
- 支持关闭引导，提供「重新显示引导」入口

#### 2.2.9 自定义主题 / 视图布局可视化
- 支持切换浅色 / 深色主题，适配不同使用场景
- 允许用户自定义视图布局（如调整原始 / 修复 SQL 面板的宽度比例）
- 记住用户的布局偏好

## 3. 技术需求

### 3.1 技术栈
- **后端**：Java 17, Spring Boot 3.2.4, ANTLR4 4.13.1
- **前端**：HTML5, CSS3, JavaScript, Chart.js
- **数据库**：YashanDB 1.9.3
- **构建工具**：Maven 3.8+

### 3.2 性能要求
- 批量处理进度、统计仪表盘等实时更新的组件需做节流 / 防抖处理，避免页面卡顿
- 支持处理大型SQL文件（最大10MB）
- 响应时间：单条SQL处理不超过2秒，批量处理10个文件不超过30秒

### 3.3 兼容性要求
- 所有可视化组件需适配 PC 端主流浏览器（Chrome/Firefox/Edge）
- 响应式布局，适配不同屏幕尺寸
- 兼容YashanDB 1.9.3及以上版本

### 3.4 安全要求
- 输入验证：防止SQL注入和XSS攻击
- 文件上传安全：验证文件类型和大小
- 错误处理：避免敏感信息泄露

## 5. 支持的修复规则

### 5.1 函数修复
- DECODE → CASE
- NVL → COALESCE
- SYSDATE → CURRENT_TIMESTAMP
- TO_DATE → TO_TIMESTAMP
- TO_CHAR → CAST
- ROWNUM → ROW_NUMBER() OVER ()
- RANK → RANK
- DENSE_RANK → DENSE_RANK
- LEAD → LEAD
- LAG → LAG
- LISTAGG → STRING_AGG
- REGEXP_SUBSTR → SUBSTRING
- REGEXP_REPLACE → REGEXP_REPLACE

### 5.2 语法修复
- DUAL表 → (SELECT 1 AS dummy)
- ROWNUM → LIMIT 1
- ROWID → ctid
- 序列.NEXTVAL → nextval('序列')
- 序列.CURRVAL → currval('序列')
- TIMESTAMP字面量 → '...'::timestamp
- INTERVAL字面量 → '...'::interval '...'
- DATE字面量 → '...'::date

### 5.3 类型修复
- VARCHAR2 → VARCHAR
- NUMBER → INTEGER/NUMERIC
- DATE → TIMESTAMP
- TIMESTAMP → TIMESTAMP
- CLOB → TEXT
- BLOB → BYTEA

## 6. 系统架构

### 6.1 核心模块
- **core**：核心解析和修复引擎，使用ANTLR4进行SQL解析
- **service**：业务逻辑服务，处理SQL扫描、转换和验证
- **controller**：API接口控制器，处理HTTP请求
- **infra**：基础设施工具，如JDBC连接和文件读取
- **dto**：数据传输对象，定义API请求和响应格式

### 6.2 数据流
1. 用户输入SQL或上传文件
2. 前端发送请求到后端API
3. 后端使用ANTLR4解析SQL
4. 应用修复规则转换SQL
5. 验证修复后的SQL
6. 返回结果给前端展示

## 11. 风险评估

### 11.1 技术风险
- ANTLR4语法解析可能无法覆盖所有Oracle SQL语法
- YashanDB版本更新可能导致兼容性问题

### 11.2 业务风险
- 复杂SQL语句的修复可能需要人工干预
- 性能瓶颈可能影响用户体验

### 11.3 应对措施
- 持续更新ANTLR4语法规则
- 定期测试与最新YashanDB版本的兼容性
- 优化性能，增加批处理能力
- 提供人工干预的接口和文档
